# Why the Stylometry audit consumed zero credits

## Executive finding

The Stylometry audit completed successfully, but consumed **0 model credits**.
The session meter showed **2300 credits before and after** the run. This was
not evidence that FORGE skipped the audit. It is evidence that this run stayed
on FORGE's deterministic analysis path and did not invoke a model-backed agent
or orchestrator.

The distinction is important:

- **Credits** measure model-backed work in the external session.
- **Elapsed time** measures local computation performed by FORGE.
- **Audit completeness** is measured by coverage, scope boundaries, findings,
  disposition, and seal verification—not by credit consumption.

## What FORGE actually did

The run performed local, deterministic work:

1. It discovered 241 files and classified the repository.
2. It analyzed 31 Python modules within the active scope.
3. It ran the deterministic Archaeologist, Bug Investigator, Security Auditor,
   and Integrity Inspector paths.
4. The Integrity Inspector emitted six AST observations:
   five unversioned-serialization findings and one decision-adjacent-float
   finding.
5. It preserved the scope boundary instead of presenting the result as a
   complete audit.
6. It generated JSON and HTML artifacts, created the verification seal, and
   verified the CRONOS chain.

The result was intentionally:

```text
ABSTAIN_INSUFFICIENT_SCOPE
```

Only 31 of 241 discovered files were analyzed. The omitted boundary included
policy-excluded files, binary or unreadable files, non-Python files, and
modules outside `CONNECTED_ALIVE`. The six findings remain useful evidence,
but they do not justify a complete-repository claim.

## Why no LLM credit was needed

FORGE's current decision path uses Python AST inspection, deterministic
classification, rule-based detectors, local metrics, and cryptographic
artifact verification. Those operations consume CPU time and filesystem I/O,
not model credits. The CRONOS trace confirms that no agent or orchestrator
model routing was configured for this run.

This is the intended architectural property: the audit decision does not
depend on an LLM reading every file. A model can be added later for optional
narration or explanation, but that must remain outside the finding, seal, and
disposition path.

## What the 0.440418-second value means

`0.440418` is the local audit duration recorded by CRONOS. It is **not** a
credit price and must not be presented as “0.440418 credits”. Likewise, the
session's unchanged `2300 credits` is the observed external meter result, not
a measurement generated by FORGE itself.

## Current accounting limitation

This run produced `metrics.json`, CRONOS trace data, and all report artifacts,
but did not produce a `run-cost.json` sidecar. Consequently, the sealed FORGE
artifacts can prove:

- deterministic model routing was empty;
- local elapsed time;
- files analyzed and omitted;
- findings, disposition, and evidence boundaries;
- seal and chain integrity.

They cannot independently prove a numeric credit delta. The authoritative
credit observation for this run is the external session meter: 2300 before and
2300 after. A future accounting contract should emit a `run-cost.json` with
explicit fields for model calls, credits before/after when available, credits
consumed when available, deterministic CPU time, and an “unknown/not supplied”
state rather than inferring credits from elapsed time.

## Conclusion

The run was cheap because FORGE analyzed the repository with deterministic
local mechanisms and invoked no model-backed routing. It was not a null run:
it generated six findings, correctly abstained on incomplete scope, sealed the
artifacts, and verified the CRONOS chain. The next observability improvement is
to make the cost sidecar explicit so future demos can show both facts together:
**model credits consumed** and **deterministic audit work performed**.
